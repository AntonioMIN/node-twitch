    TwitchApi

    Class to control access to the Twitch api.

    Constructor

    new TwitchApi(config)

    Initialize the api.

    Parameters:
    Name Type Description
    config Object

    A configuration object containing your client_id and client_secret, as well as an access_token and refresh_token.

    Properties
    Name Type Attributes Description
    client_id string

    Your client id.
    client_secret string

    Your client secret.
    scopes Array.<string> <optional>

    The scopes that your application requires. Only needed when using a user access token.
    access_token string <optional>

    The access token from an authenticated user.
    refresh_token string <optional>

    The refresh token from an authenticated user.
    isApp bool <optional>

    A boolean value that determines whether or not the api should fetch an app access token. When using this option, you are only able to access public user information.
    Properties
    Name Type Description
    user Object

    Object containing information about the currently authenticated user.

    Properties
    Name Type Description
    broadcaster_type string

    User's broadcaster type: "partner", "affiliate", or "".
    description string

    User's channel description.
    display_name string

    User's display_name.
    email string

    User's email address. Is only included if scope user:read:email was included in the authentication scopes.
    id string

    User's ID.
    login string

    User's login name.
    offline_image_url string

    URL of the user's offline image.
    profile_image_url string

    URL of the user's profile image.
    type string

    User's type: "staff", "admin", "global_mod", or "".
    view_count number

    Total number of views of the user's channel. Does not update overtime.

    (async, inner) getBitsLeaderboard(options, callbackopt)

    Get the bits leaderboard of a user or top users.

    Parameters:
    Name Type Attributes Description
    options Object

    The options for the request.

    Properties
    Name Type Attributes Description
    count number <optional>

    Number of results to be returned. Maximum: 100. Default: 10.
    period string <optional>

    Time period over which data is aggregated (PST time zone). This parameter interacts with started_at. Valid values are given below. Default: "all". For more information visit the official api docs.
    started_at string <optional>

    Timestamp for the period over which the returned data is aggregated. Must be in RFC 3339 format. Ignored if period is "all".
    user_id string <optional>

    ID of the user whose results are returned; i.e., the person who paid for the Bits. If user_id is not provided, the endpoint returns the Bits leaderboard data across top users (subject to the value of count).
    callback apiCallback <optional>

    The callback function.

    (async, inner) getStreams(optionsopt, callbackopt)

    Get one or more live streams.

    Parameters:
    Name Type Attributes Description
    options Object <optional>

    An options object used to create the request.

    Properties
    Name Type Attributes Description

    Cursor for forward pagination: tells the server where to start fetching the next set of results, in a multi-page response. The cursor value specified here is from the pagination response field of a prior query.

    Cursor for backward pagination: tells the server where to start fetching the next set of results, in a multi-page response. The cursor value specified here is from the pagination response field of a prior query.
